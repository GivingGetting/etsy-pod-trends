# 数据更新指南

所有趋势数据存放在 `data.js` 文件中，手动编辑即可更新网站内容，无需重启或构建。

---

## 数据结构总览

```
trendData
├── categories        品类数据（T恤、马克杯、贴纸等）
├── styles            设计风格数据
├── colors            流行颜色
├── seasonal          季节性事件和月度趋势
├── rankings          三个排行榜
├── chartData         折线图数据（近6个月趋势）
└── pieData           饼图数据（市场份额）
```

---

## 一、更新品类数据

**位置**：`data.js` → `categories` 数组

每个品类的字段说明：

```js
{
    id: 'apparel',          // 唯一标识符，不要修改
    name: 'T恤设计',        // 显示名称
    icon: '👕',             // 表情图标
    growth: 23,             // 增长率（%），正数=增长，负数=下降
    monthlyVolume: '12.5K', // 月搜索/销售量（显示用，字符串）
    popularity: 85,         // 热度评分（0-100）
    subcategories: [        // 子品类数组
        { name: '图案T恤', growth: 28, popularity: 90 },
        ...
    ]
}
```

**如何添加新品类**：

```js
// 在 categories 数组末尾添加
{
    id: 'pillow',           // 新的唯一 id
    name: '抱枕',
    icon: '🛋️',
    growth: 30,
    monthlyVolume: '3.5K',
    popularity: 60,
    subcategories: [
        { name: '定制照片枕', growth: 35, popularity: 65 },
        { name: '文字抱枕', growth: 25, popularity: 55 }
    ]
}
```

## 二、更新设计风格

**位置**：`data.js` → `styles` 数组

```js
{
    name: '极简主义',       // 风格名称
    popularity: 92,         // 热度（0-100）
    growth: 18,             // 增长率（%）
    description: '...'      // 风格描述（显示在详情卡片）
}
```

**建议更新频率**：每季度一次，根据 Etsy 实际销售数据调整 `popularity` 和 `growth`。

## 三、更新流行颜色

**位置**：`data.js` → `colors` 数组

```js
{
    name: '薰衣草紫',       // 颜色名称
    hex: '#B57EDC',         // 十六进制颜色值（决定色块显示颜色）
    popularity: 88          // 热度（0-100）
}
```

**查找 Hex 色值**：在 Google 搜索颜色名称，或用 [coolors.co](https://coolors.co) 取色。

## 四、更新季节性数据

### 4.1 时间线事件

**位置**：`data.js` → `seasonal.timeline`

```js
{ month: '2月', event: '情人节', active: false, upcoming: true }
```

| 字段 | 说明 |
|------|------|
| `active: true` | 当前正在进行的节日，高亮显示 |
| `upcoming: true` | 即将到来，特别标注 |
| 两者都 `false` | 普通月份 |

**每月初更新**：将当前月设为 `active: true`，下个月设为 `upcoming: true`，其余设为 `false`。

### 4.2 即将到来的节日

**位置**：`data.js` → `seasonal.upcomingEvents`

```js
{
    name: '情人节',
    date: '2月14日',
    daysUntil: 31,      // 距今天数（手动填写）
    icon: '💕'
}
```

**注意**：`daysUntil` 需要手动计算并更新，建议每月更新一次。

### 4.3 月度销售趋势

**位置**：`data.js` → `seasonal.monthlyTrends`

数值范围 0-100，代表该品类在当月的相对热度：

```js
{ month: '12月', apparel: 95, home: 95, accessories: 92, stickers: 88 }
```

## 五、更新排行榜

**位置**：`data.js` → `rankings`

包含三个榜单，每个最多显示 5 条：

```js
rankings: {
    topCategories: [        // 热门品类榜（按增长率排序）
        { name: '贴纸', value: '+32%' },
        ...
    ],
    topStyles: [            // 热门风格榜（按热度排序）
        { name: '极简主义', value: '92%' },
        ...
    ],
    fastestGrowing: [       // 增长最快榜
        { name: '宠物用品', value: '+45%' },
        ...
    ]
}
```

## 六、更新折线图（近期趋势）

**位置**：`data.js` → `chartData`

显示最近 6 个月的品类趋势对比：

```js
chartData: {
    labels: ['8月', '9月', '10月', '11月', '12月', '1月'],  // 月份标签
    datasets: {
        apparel:     [65, 70, 85, 80, 95, 88],  // 服装类
        home:        [60, 75, 88, 85, 95, 82],  // 家居类
        accessories: [55, 65, 90, 88, 92, 78]   // 配件类
    }
}
```

**每月滚动更新**：删除最旧的一个月数据，在末尾加入最新月份。

---

## 七、更新饼图（市场份额）

**位置**：`data.js` → `pieData`

```js
pieData: {
    labels: ['服装类', '贴纸文具', '家居装饰', '配件类', '其他'],
    values: [30, 25, 20, 15, 10]   // 百分比，所有值加起来应等于 100
}
```

## 八、更新后发布

修改 `data.js` 保存后，执行以下命令推送到线上：

```bash
git add data.js
git commit -m "更新趋势数据 - 2026年3月"
git push
```

Vercel 会在 **30 秒内** 自动重新部署，刷新 https://daystarwin.com 即可看到更新。

| [Pinterest 趋势](https://trends.pinterest.com) | 设计风格和颜色灵感 | ✅ |

## 快速检查清单

每次更新前确认：

- [ ] `popularity` 和 `growth` 数值基于实际数据，而非猜测
- [ ] `pieData.values` 加起来等于 100
- [ ] `seasonal.timeline` 只有一个月设为 `active: true`
- [ ] `chartData.labels` 和 `datasets` 中每个数组长度一致（都是 6 个）
- [ ] 推送后刷新网站验证显示正常
